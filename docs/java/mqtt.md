# mqtt

## MQTT（Message Queuing Telemetry Transport）
**定义：** 一种基于发布/订阅模式的轻量级消息协议，运行于TCP/IP之上，专为物联网场景设计。

**特点：**
- 低开销：固定头部仅2字节。
- 三种QoS等级：0（最多一次）、1（至少一次）、2（恰好一次），适应不同可靠性需求。
- 持久会话：允许设备离线时保存订阅和未确认消息，上线后恢复。
- 支持遗愿消息：设备异常断开时，Broker可自动发布预设消息。

## MQTT Broker
**定义：** MQTT协议的**服务器端**，负责接收所有客户端（设备或后端服务）的连接，处理消息的发布与订阅。

**核心职责：**  
- 维护**主题树**，根据订阅关系路由消息。
- 管理客户端会话、QoS保障、消息持久化。
- 提供安全认证、访问控制、集群能力。

MQTT Broker通常作为一个集群，承载所有MQTT的通信。

如何搭建？或者有什么成熟的实现方案：
- **EMQX**：适合企业级生产环境，也适合个人项目
- **Eclipse Mosquitto**：适合本地开发、单元测试

## 设备

分为【支持MQTT】的设备和【使用私有二进制协议】的设备

## Eclipse Paho

Eclipse Paho 是Eclipse基金会维护的**开源MQTT客户端实现项目**，支持多种编程语言。  

在JAVA项目中添加Paho作为依赖，便可使项目变成一个**MQTT客户端**，与MQTT Broker（服务端）进行交互。

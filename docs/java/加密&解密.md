# 加密算法



根据用途，主要分为三大类：
- **对称加密**（加密和解密使用相同密钥）
- **非对称加密**（使用公钥和私钥对）和
- **哈希算法**（不可逆的摘要）。


## 1. 对称加密算法

常见的有 **AES**、**DES**、**3DES** 等，目前 **AES** 是主流标准。

使用场景：
- **网络通信加密**：如**HTTPS**、**VPN**等。防止数据经过公网时被窃听。
- **静态数据加密**：加密本地文件和磁盘。

## 2. 非对称加密

非对称加密使用一对密钥：公钥（公开）和私钥（保密）。常用于密钥交换、数字签名。
常见的有：
- **RSA**
- **ECC**（椭圆曲线）
- ...

使用场景：
- **安全连接握手**：
  - 在 SSL/TLS（即HTTPS的底层协议）握手阶段，客户端和服务器使用 RSA 或 ECC 算法来安全地协商出一个临时的对称密钥，用于后续的通信加密。
  - 像GIT就是为了通过 SSH 协议安全地连接远程仓库（如 GitHub、GitLab、Gitee），实现免密码的身份验证。
- **数字签名**：
  - 验证软件发布者的身份和软件的完整性。开发者用其私钥对安装包的哈希值进行签名（如使用 RSA 或 ECDSA），你的电脑用开发者的公钥验证签名
- **身份认证**：
  - 证书上有公钥，需要用你的私钥验证是否匹配，从而确认你的身份。

## 3. 哈希算法（不可逆）

哈希算法将任意长度的数据映射为固定长度的摘要，用于验证数据完整性、存储密码等。
常见的有：
- **SHA-256**：输出64 个十六进制字符（256 位）
- **MD5**（已不安全）：
- ...

使用场景：
- **密码存储**：数据库一般不存储明文密码，而是存密码的哈希值。（一般都用SHA-256）
- **文件完整性校验**：若你计算的文件的哈希值与来源地提供的不一致，则文件可能在下载过程中损坏或被篡改。
- **数字签名的辅助**：由于非对称加密速度慢，不会直接对整个大文件签名，而是先对文件进行哈希计算，然后再对这个短得多的哈希值进行签名，大大提高了效率

## 4. 消息认证码（HMAC）

HMAC 结合哈希算法和密钥，用于验证消息的完整性和来源。常用于 API 签名。
常见的有：
- **HMAC-SHA256**

使用场景：
- API 请求签名
- JWT签名
- 网络协议中的消息认证

## 5. 编码算法（非加密）

虽然不属于加密，但常被误认为加密。如：
- **Base64**：`将二进制数据转换为可打印 ASCII 字符，不提供安全性，仅用于传输。
- **URL编码**：：用于将 URL 中的非 ASCII 字符或特殊字符（如空格、中文、&、# 等）转换成 % 后跟两位十六进制数的形式。

使用场景：
- 用文本形式保存二进制数据：比如一些图片、视频，或者可执行文件
